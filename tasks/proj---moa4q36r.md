## Goal

Сделать локальный таск-менеджер как рабочую замену Notion для личного планирования, исполнения задач через Codex и интеграции в личную экосистему

Критерии результата:
- проектами можно управлять через UI и через API
- состояние проектов хранится локально и предсказуемо
- Codex умеет создавать, менять, двигать и комментировать задачи
- система интегрируется с Google Calendar и готова к связке с Obsidian

## Context

- Приложение построено на Next.js 16 и уже включает board/status view, priority view, список всех проектов и Gantt.
- Каноническое состояние хранится в data/task-manager.json, а tasks/*.md используются как заметки и могут отставать от JSON.
- Основной интерфейс для агентной работы: POST /api/agent с операциями list, get, create, update, delete, read_notes, write_notes, log_progress.
- Дополнительные route уже есть для focus, progress, tags, google-calendar и drive.
- Для Codex создан отдельный superpowers skill task-manager-agent, который работает API-first и не опирается на прямые правки state-файлов по умолчанию.
